1为什么需要人格系统

在加入人格系统之前，所有 agent 的 LLM 行为是完全同质的。 即使给 agent 配置了不同的 RoleType，面对相同任务时，LLM#3 的选槽逻辑、 ADM 的决策风格、记忆权重的分配方式都一模一样——哪怕两个 agent 的"能力"在数值上相同， 它们的行为也不应该完全一样。

加入前 — 行为同质
  • LLM#3 选槽：纯粹按电量/位置匹配，没有个体偏好
  • ADM 决策：所有 agent 决策风格相同，面对风险的处理方式一致
  • Memory 权重：所有类型的记忆 importance 按统一规则打分
  • Reflection 阈值：所有 agent 反思频率相同
  • 结果：大量 agent 抢同一个角色、决策结果雷同
加入后 — 行为分化
  • LLM#3 选槽：高尽责性 → 倾向 Perimeter，高开放性 → 倾向 Scout
  • ADM 决策：高神经质 agent 倾向先 Observe，高开放性 agent 尝试非常规路径
  • Memory 权重：各维度对不同类型记忆的 importance 乘数不同
  • Reflection 阈值：高神经质 agent 反思更频繁（阈值 150→100）
  • 结果：稳定的角色专业化涌现，跨任务行为一致性提升
核心设计原则：人格是注入 LLM prompt 的语义信号，不是硬编码的规则。 LLM 依然有自主权决定是否参考——但 prompt 中存在这些信号后，LLM 的行为会有统计上的一致偏移， 并在 thought 字段中可见地解释了自己是否参考了人格偏好，便于调试。
2PersonalityProfile — 数据结构

PersonalityProfile 是一个 [Serializable] 数据类， 定义在 PersonalitySystem.cs 同文件中，在 Inspector 中配置。 分为三层：大五人格维度、领域特化描述、运行时衍生倾向。

2.1 大五人格维度（Inspector 配置，Range [0,1]）

字段 高值效果（>0.7） 影响哪里 记忆重要度乘数
openness
开放性		 好奇探索，愿意试非常规路径 LLM#3 偏好 Scout/Explorer；ADM 可尝试新路径；记忆偏重观察 Observation ×1.1
conscientiousness
尽责性		 系统化执行，严格按步骤 LLM#3 偏好 Perimeter/Guard；ADM 每步完成后确认再推进 Plan ×1.1
extraversion
外向性		 主动协调，广播状态 LLM#3 偏好 Coordinator；ADM 主动 Signal/Whiteboard （不单独加权）
agreeableness
宜人性		 配合队友优先，服从指令 LLM#3 偏好 Supporter；ADM 优先协调而非独立行动 （不单独加权）
neuroticism
神经质		 对失败敏感，更频繁反思 ADM 优先保守方案；失败记忆权重×1.2；L2 反思阈值 150→100 Decision ×1.2

2.2 领域特化描述（Inspector 配置，字符串数组）

这三个字段以自由文本数组形式直接出现在 LLM prompt 中， 补充大五维度无法精确描述的具体行为倾向。

coreValues — 核心价值观

LLM 在做决策时将这些价值观作为优先级依据。

"安全第一", "团队优先", "效率"
habits — 行为习惯

LLM 在生成 nextActions 时会倾向遵循。

"先观察后行动", "按步骤执行"
dislikes — 厌恶/回避项

LLM 在选槽和行动时会尽量避免触发这些条件。

"冲突", "模糊指令", "独自行动"

2.3 运行时衍生字段（[HideInInspector]，由 Initialize() 计算）

字段计算公式含义
riskTolerance openness×0.5 + (1−neuroticism)×0.5 风险容忍度。高开放+低神经质 → 愿意接受不确定结果
cooperationBias agreeableness×0.6 + extraversion×0.4 协作偏好。高宜人+高外向 → 强烈团队优先
3PersonalitySystem — 六个核心方法
🎭
GetRolePreferenceHint()
生成角色偏好提示字符串，注入 LLM#3 选槽 prompt。各维度>0.7 时追加对应偏好描述。
→ PlanningModule
⚡
GetDecisionStyleHint()
生成行动风格描述，注入 ADM Rolling Prompt。神经质/开放性/尽责性/协作偏好均有对应描述。
→ ActionDecisionModule
⚖️
GetImportanceModifier(kind)
根据记忆类型和人格维度返回乘数（1.0~1.2）。在 StoreMemory 时修正 importance 字段。
→ MemoryModule
🏷️
GetPersonalityTag()
返回突出维度的标注字符串（如 [高尽责性-0.85]），写入 Memory.personalityContext。
→ Memory.personalityContext
🔁
ShouldTriggerEarlyReflection()
神经质>0.7 时返回 true。触发时 ReflectionModule 将 L2 阈值从 150 降为 100。
→ ReflectionModule
🔍
GetPreferredRoles()
返回人格偏好的角色名列表，追加到 MemoryQuery.preferredRoles，记忆检索时对应角色加权。
→ MemoryQuery

方法逻辑速查

// GetRolePreferenceHint() — 高维度→追加偏好文字，多条用"；"连接
if (conscientiousness > 0.7f) → "倾向选择边界管控或系统性角色(如Perimeter/Guard)"
if (openness          > 0.7f) → "倾向选择侦察或探索角色(如Scout/Explorer)"
if (extraversion      > 0.7f) → "倾向选择协调或领导角色(如Coordinator)"
if (agreeableness     > 0.7f) → "倾向选择支援角色，优先配合队友需求"

// GetDecisionStyleHint() — 多维度叠加，换行连接
if (neuroticism        > 0.7f) → "谨慎行事，优先规避风险，遇不确定情况选保守方案"
elif (neuroticism      < 0.3f) → "执行稳健，失败后迅速调整继续推进，不过度纠结"
if (openness           > 0.7f) → "可尝试非常规路径，允许在标准流程之外寻找更优方案"
if (conscientiousness  > 0.7f) → "按步骤系统化执行，每步完成后确认状态再推进下一步"
if (cooperationBias    > 0.6f) → "优先与队友协调，主动通过信号广播自身状态和意图"

// GetImportanceModifier(kind) — switch 表达式
Decision   + neuroticism        > 0.6f  →  1.2f  // 失败决策加权
Plan       + conscientiousness  > 0.7f  →  1.1f  // 计划加权
Observation + openness         > 0.7f  →  1.1f  // 观察加权
_                               →  1.0f  // 不修改

MemoryModule.cs
MODIFY

串联方式

IntelligentAgent.InitializeAgent() 在初始化末尾调用 memoryModule.SetPersonalitySystem(ps)，将人格系统注入。 之后每次 StoreMemory() 都会自动在 Add() 前调用 InjectPersonality(m)， 所有写入记忆的调用方无需任何改动。

StoreMemory(m) 调用
→
GetImportanceModifier(m.kind)
→
m.importance × modifier
→
Clamp01 写回
同一次调用
→
GetPersonalityTag()
→
m.personalityContext = "[高尽责性-0.85]"

ScoreMemory 中 preferredRoles 加权

当 MemoryQuery.preferredRoles 非空时（由 PlanningModule 在 BuildPlanningContext 中可选填入）， ComputeNormalizedRelevance() 对 tags 中包含偏好角色名的记忆额外 +0.10f/条，上限 0.20f， 使历史上担任偏好角色的记忆在检索时优先被召回。

新增字段

  • Memory.personalityContext — 记忆创建时的人格标注
  • MemoryQuery.preferredRoles — 检索时偏好角色名集合

🔁
ReflectionModule
ReflectionModule.cs
MODIFY

串联方式

EnsureDependencies() 中 GetComponent<PersonalitySystem>()， 在首次获取到 PersonalitySystem 后立即判断是否需要调整阈值 （而非在 Start 中固定调整，因为 EnsureDependencies 可能在 Start 之外被调用）。

阈值调整逻辑

// EnsureDependencies() 末尾
if (_personalitySystem != null && _personalitySystem.ShouldTriggerEarlyReflection()
    && memoryModule != null)
{
    // neuroticism > 0.7 → 降低 L2 反思触发阈值
    memoryModule.reflectionImportanceThreshold = 100f;  // 默认 150
}

阈值含义

每次 StoreMemory 时将 importance×10 累加到累积器。 累积器满阈值时触发一次 L2 跨事件反思（LLM 调用）。

人格阈值约需多少条高权重记忆触发一次
低/中神经质（默认） 150 约 15 条 importance=1.0 的记忆
高神经质（>0.7） 100 约 10 条，反思频率提升 50%
注意：更频繁的反思意味着更多 LLM token 消耗。 这是人格特征的有意体现——"过度思考"是高神经质人格的真实特征。 若 token 预算有限，可将 neuroticism 阈值从 0.7 调高。
5初始化流程（IntelligentAgent）

人格系统的串联由 IntelligentAgent.InitializeAgent() 统一驱动， 其余模块（PlanningModule、ADM、ReflectionModule）通过 GetComponent 自取引用， 不需要 IntelligentAgent 手动传递。

Step 1 — GetComponent / AddComponent
var personalitySystem = GetComponent<PersonalitySystem>();
if (personalitySystem == null)
    personalitySystem = gameObject.AddComponent<PersonalitySystem>(); // 未挂载时自动添加默认人格
if (string.IsNullOrWhiteSpace(personalitySystem.Profile.agentId))
    personalitySystem.Profile.agentId = Properties.AgentID; // 同步 AgentID
Step 2 — Initialize()（计算衍生值）
personalitySystem.Initialize();
// 计算 riskTolerance = openness×0.5 + (1−neuroticism)×0.5
// 计算 cooperationBias = agreeableness×0.6 + extraversion×0.4
Step 3 — 注入 MemoryModule
var memoryModule = GetComponent<MemoryModule>();
memoryModule?.SetPersonalitySystem(personalitySystem);
// 此后所有 StoreMemory 调用均自动注入人格信息
Step 4 — 其余模块在各自 Start() 中自取
// PlanningModule.Start():
_personalitySystem = GetComponent<PersonalitySystem>();

// ActionDecisionModule.Start():
_personalitySystem = GetComponent<PersonalitySystem>();

// ReflectionModule.EnsureDependencies():
_personalitySystem = GetComponent<PersonalitySystem>();
if (_personalitySystem.ShouldTriggerEarlyReflection())
    memoryModule.reflectionImportanceThreshold = 100f;
挂载方式：在每个 agent GameObject 的 Inspector 中添加 PersonalitySystem 组件并填写 Profile 字段即可。 若不挂载，InitializeAgent 会自动添加一个所有维度为默认值（0.5）的中性人格， 不影响其他模块正常运行。

数据流总览

IntelligentAgent.InitializeAgent()
    │
    ├─ PersonalitySystem.Initialize()   ← 计算衍生值
    │       │
    │       └─→ MemoryModule.SetPersonalitySystem()
    │               │
    │               └─ StoreMemory() 时自动调用 InjectPersonality()
    │                       ├─ importance × GetImportanceModifier(kind)
    │                       └─ personalityContext = GetPersonalityTag()
    │
    ├─ PlanningModule（自取引用）
    │       └─ RunLLM3() prompt += GetRolePreferenceHint()
    │
    ├─ ActionDecisionModule（自取引用）
    │       └─ BuildRollingPrompt() += "## 行动风格\n" + GetDecisionStyleHint()
    │
    └─ ReflectionModule（自取引用）
            └─ ShouldTriggerEarlyReflection() → true
                    └─ memoryModule.reflectionImportanceThreshold = 100f

7完整例子：一次选槽 → 执行 → 记忆的全流程

以 Hawk-1（高尽责性 0.85，低神经质 0.30） 执行"拦截北门入侵者"任务为例， 展示人格系统在每个关键节点产生的实际影响。

阶段 1 — 初始化
// IntelligentAgent.InitializeAgent()
personalitySystem.Initialize();
// → riskTolerance  = 0.55×0.5 + (1-0.30)×0.5 = 0.625
// → cooperationBias = 0.70×0.6 + 0.40×0.4 = 0.580
memoryModule.SetPersonalitySystem(personalitySystem);
// MemoryModule 已就绪，后续所有写入自动注入人格信息
阶段 2 — LLM#3 选槽

RunLLM3() 调用 GetRolePreferenceHint()，追加到 prompt： 

[人格倾向] 倾向选择边界管控或系统性角色(如Perimeter/Guard)；
倾向选择支援角色，优先配合队友需求
请在 thought 字段中说明是否参考了上述人格倾向。

LLM 返回的 thought 中出现了："人格倾向系统性角色，位置靠近南侧， s2(Perimeter封锁南侧出口)与人格特征和位置高度匹配" → 选 s2。

阶段 3 — RememberSlotSelection 写入记忆
// MemoryModule.StoreMemory() 内自动调用 InjectPersonality()

// ① importance 修正
modifier = GetImportanceModifier(AgentMemoryKind.Plan)
//   conscientiousness=0.85 > 0.7 → modifier = 1.1f
m.importance = 0.65f × 1.1f = 0.715f

// ② 人格标注写入
m.personalityContext = "[高尽责性-0.85][高宜人性-0.70]"
阶段 4 — ADM Rolling Loop 执行

BuildRollingPrompt() 在"步骤目标"后追加"行动风格"段： 

## 行动风格
按步骤系统化执行，每步完成后确认状态再推进下一步
优先与队友协调，主动通过信号广播自身状态和意图

LLM 在 thought 中解释："按系统化步骤：先飞至南侧出口建立监视点， 再写 IntentAnnounce 告知队友。" → nextActions: [MoveTo("南侧出口"), WriteWhiteboard("IntentAnnounce")]

阶段 5 — 任务成功后 RememberMissionOutcome
m.importance  = 0.94f  // Outcome 类型不在修正规则内，使用原始值
m.personalityContext = "[高尽责性-0.85][高宜人性-0.70]"
m.tags        = ["mission_success", "Perimeter", "slot_selection"]
阶段 6 — 下次类似任务检索时

BuildPlanningContext 构建 MemoryQuery，可选填入 preferredRoles： 

query.preferredRoles = _personalitySystem.GetPreferredRoles();
// → ["Perimeter", "Guard", "Supporter"]

// ScoreMemory 中：tags 含 "Perimeter" → +0.10f 额外加分
// 上次成功的 Perimeter 任务记忆会被优先召回
涌现效果：连续任务后，Hawk-1 会稳定地强化"担任 Perimeter 角色"的偏好—— 这是人格系统带来的角色专业化自然涌现，无需硬编码。
