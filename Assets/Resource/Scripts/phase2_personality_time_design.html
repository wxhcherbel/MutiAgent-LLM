1记忆模块诊断：当前问题与价值评估

1.1 一般应该记什么？

记忆的核心价值是"改变未来决策"。一条记忆只有在被检索时能让 LLM 做出不同（且更好）的判断，它才有意义。衡量标准是：如果没有这条记忆，LLM 会做错什么？

✅值得记忆的内容
类型示例作用
失败动作 + 原因"从东侧接近被目标发现"避免重蹈覆辙
环境状态观测"北门14:30人员密集"时空感知积累
角色历史成效"担任Scout完成率87%"LLM#3选槽偏好
约束推理依据"充电桩共享→需C3互斥"同类任务复用
反思洞察（isProceduralHint）"高空观察效率>低空扫描"跨任务通用规则
协作模式"与UAV-2搭档完成率高"社会关系积累
❌不值得记忆的内容
类型原因
成功的平凡动作
"MoveTo执行成功"		不提供差异化信息，稀释失败信号
重复相同结论占用容量，降低检索精度
无上下文的状态码"步骤1完成"缺乏条件信息，无法泛化
执行细节（坐标等）随时间失效，且LLM从感知层可实时获取

1.2 当前记忆体系的问题

核心问题：记忆内容是客观的，但缺乏主观维度。 当前系统记录了"发生了什么"，但没有记录"在什么状态下发生的"（时间、人格、情绪）以及"我对此的解读"（不是从 thought 里提取的主观判断）。
当前记忆点实际存储内容缺什么有没有用

RememberSlotDesign
（今天刚加）		 角色列表+LLM角色设计思路 人格对角色偏好的影响 有用 —— 组长下次设计角色时参考

RememberPlanSnapshot 步骤列表+LLM拆解推理 时间段（下午的计划vs晚上的计划不同） 中等 —— 无时间维度时跨时段检索噪声大

2SimulatedTimeModule.cs NEW —— 时间感知模块

让每个 agent 拥有"现在是什么时间"的感知，并在关键时间节点（换日、换时段）触发事件，为后续的自主目标生成（Phase 3）做铺垫。

2.1 枚举与数据结构

public enum TimePeriod
{
    Night     = 0,  // 21:00 - 07:00  夜间，能见度低，建议减少主动行动
    Morning   = 1,  // 07:00 - 12:00  上午，状态最佳，适合侦察/搜索
    Noon      = 2,  // 12:00 - 14:00  午间，人员流动高峰，谨慎行动
    Afternoon = 3,  // 14:00 - 18:00  下午，光线良好，适合全面任务
    Evening   = 4,  // 18:00 - 21:00  傍晚，光线减弱，需调整侦察策略
}

2.2 核心字段说明

字段类型说明注入目标
timeScalefloat 时间倍率。默认 60f 意为 1 现实秒 = 1 仿真分钟。可在 Inspector 中动态调整。 内部驱动
startHourfloat 仿真开始时的小时数（0-24）。默认 8.0（早上8点），决定 agent 的"首日"起点。 初始化
CurrentSimTimefloat (Property) 当前仿真时间（秒），由 Update() 持续累加。是所有时间计算的基准。 MemoryModule.StoreMemory
CurrentHourfloat (Property) 0-24 小时制，精确到分钟。例如 14.5 = 14:30。 LLM Prompt 注入
CurrentDayint (Property) 仿真天数，从第1天开始计。触发 OnNewDay 事件使用。 MemoryModule 字段 simDay
CurrentPeriodTimePeriod (Property) 当前时段枚举，由 CurrentHour 换算。时段切换时触发 OnPeriodChanged。 MemoryModule 字段 timePeriod
FormattedTimestring (Property) 格式化时间字符串，如"第3天 14:30 [下午]"，用于直接插入 LLM prompt。 所有 LLM Prompt
OnNewDayevent Action 每当 CurrentDay 增加时触发。Phase 3 的 AutonomousGoalGenerator 订阅此事件以触发新一天的目标规划。 AutonomousGoalGenerator（Phase 3）
OnPeriodChangedevent Action<TimePeriod> 时段切换时触发。ADM 可订阅以调整行动策略（如从下午切换到傍晚后提高谨慎度）。 ActionDecisionModule

2.3 核心方法说明

public class SimulatedTimeModule : MonoBehaviour
{
    // ── 配置 ─────────────────────────────────────────────────────
    [SerializeField] public float timeScale    = 60f;  // 1秒现实 = 1分钟仿真
    [SerializeField] public float startHour   = 8f;   // 仿真从早上8点开始

    // ── 事件 ─────────────────────────────────────────────────────
    public event Action              OnNewDay;        // Phase 3 使用
    public event Action<TimePeriod> OnPeriodChanged; // ADM 订阅

    // ── 状态属性 ──────────────────────────────────────────────────
    public float      CurrentSimTime  { get; private set; }
    public float      CurrentHour     { get; private set; }
    public int        CurrentDay      { get; private set; }
    public TimePeriod CurrentPeriod   { get; private set; }

    /// <summary>供 LLM Prompt 直接插入的时间上下文字符串。</summary>
    public string FormattedTime
        => $"第{CurrentDay}天 {(int)CurrentHour:D2}:{(int)((CurrentHour % 1) * 60):D2} [{PeriodLabel(CurrentPeriod)}]";

    /// <summary>返回当前时段的行动建议，注入 LLM 帮助调整决策风格。</summary>
    public string GetActionHint()
    {
        return CurrentPeriod switch
        {
            TimePeriod.Morning   => "当前为上午，状态最佳，适合主动侦察与搜索",
            TimePeriod.Noon      => "当前为午间，人员流动高峰，谨慎识别目标",
            TimePeriod.Afternoon => "当前为下午，光线良好，适合执行完整任务",
            TimePeriod.Evening   => "当前为傍晚，光线减弱，优先选择高空观察",
            TimePeriod.Night     => "当前为夜间，能见度低，建议依赖热感/雷达传感器",
            _ => ""
        };
    }

    /// <summary>返回完整的时间上下文段落，可直接拼接进任何 LLM prompt。</summary>
    public string GetContextBlock()
        => $"当前时间：{FormattedTime}\n环境提示：{GetActionHint()}";
}

2.4 如何挂载

推荐：在 IntelligentAgent 的 Awake/Start 中通过 GetComponent<SimulatedTimeModule>() 获取引用，作为共享组件挂载在全局 GameManager 对象上（所有 agent 共享同一个时钟），或每个 agent 独立运行（如果需要不同的时间感知）。
3PersonalitySystem.cs NEW —— 工作人格模块

为每个 agent 定义稳定的个性特征，影响其在 LLM#3 选槽、ADM 行动决策和记忆重要度评估中的行为偏好。人格不随任务改变，但可以通过长期反思（Phase 3）缓慢漂移。

3.1 PersonalityProfile 数据结构

/// <summary>
/// 智能体人格档案（大五人格模型 + 领域特化）。
/// 在 Inspector 中配置初始值，运行时只读（长期漂移由 Phase 3 实现）。
/// </summary>
[Serializable]
public class PersonalityProfile
{
    public string agentId;

    // ── 大五人格维度 [0,1] ───────────────────────────────────────
    [Range(0f, 1f)] public float openness;          // 开放性：好奇心、创造力、愿意尝试新方法
    [Range(0f, 1f)] public float conscientiousness; // 尽责性：纪律性、系统性、按规则执行
    [Range(0f, 1f)] public float extraversion;      // 外向性：主动协作、主动汇报、占主导角色
    [Range(0f, 1f)] public float agreeableness;     // 宜人性：服从指令、优先配合队友
    [Range(0f, 1f)] public float neuroticism;       // 神经质：对失败的敏感程度，越高越容易触发反思

    // ── 领域特化 ─────────────────────────────────────────────────
    public string[] coreValues;  // 核心价值观，如 ["安全第一","团队优先","效率"]
    public string[] habits;      // 行为习惯，如 ["先观察后行动","优先保持队形"]
    public string[] dislikes;    // 厌恶项，如 ["冲突","模糊指令","独自行动"]

    // ── 衍生倾向（由 PersonalitySystem 计算，不在 Inspector 配置）────
    [HideInInspector] public float riskTolerance;    // 风险容忍度 = openness*0.5 + (1-neuroticism)*0.5
    [HideInInspector] public float cooperationBias;  // 协作偏好 = agreeableness*0.6 + extraversion*0.4
}

3.2 PersonalitySystem 核心方法

方法返回值说明注入目标
GetRolePreferenceHint()string 根据大五维度生成角色偏好描述。高尽责性→偏好Perimeter/Guard；高开放性→偏好Scout/Explorer；高外向性→偏好Coordinator/Leader LLM#3 prompt
GetDecisionStyleHint()string 生成行动决策风格描述。高神经质→"谨慎行事，优先规避风险"；高开放性→"可尝试非常规路径" ADM Rolling Prompt
GetImportanceModifier(AgentMemoryKind)float 根据人格调整记忆重要度。高神经质agent对失败记忆额外加权(×1.2)；高尽责性对计划记忆加权(×1.1) MemoryModule.StoreMemory
GetFullPersonalityContext()string 返回完整人格描述段落，用于 AutonomousGoalGenerator（Phase 3）和 LLM#3 完整版注入 Phase 3 自主目标生成
ShouldTriggerEarlyReflection()bool 高神经质(>0.7)时降低反思触发阈值，让agent更频繁地进行自我反思 ReflectionModule
GetPreferredRoles()string[] 根据人格维度返回偏好角色列表，用于在 BuildPlanningContext 检索中增加相关性权重 MemoryQuery.tags

3.3 GetRolePreferenceHint() 的逻辑

public string GetRolePreferenceHint()
{
    var hints = new List<string>();

    // 尽责性高 → 系统性角色（边界控制、监视）
    if (profile.conscientiousness > 0.7f)
        hints.Add("倾向选择边界管控或系统性角色(如Perimeter/Guard)");

    // 开放性高 → 探索性角色（侦察、发现）
    if (profile.openness > 0.7f)
        hints.Add("倾向选择侦察或探索角色(如Scout/Explorer)");

    // 外向性高 → 领导或协调角色
    if (profile.extraversion > 0.7f)
        hints.Add("倾向选择协调或领导角色(如Coordinator)");

    // 宜人性高 → 支援角色，配合队友
    if (profile.agreeableness > 0.7f)
        hints.Add("倾向选择支援角色，优先配合队友需求");

    return hints.Count > 0
        ? "[人格倾向] " + string.Join("；", hints)
        : "";
}

3.4 GetImportanceModifier() 的逻辑

public float GetImportanceModifier(AgentMemoryKind kind)
{
    return kind switch
    {
        // 高神经质 → 失败记忆额外加权，避免重蹈覆辙
        AgentMemoryKind.Decision when profile.neuroticism > 0.6f => 1.2f,

        // 高尽责性 → 计划记忆加权，强化流程遵从
        AgentMemoryKind.Plan when profile.conscientiousness > 0.7f => 1.1f,

        // 高开放性 → 观察记忆加权，鼓励从环境学习
        AgentMemoryKind.Observation when profile.openness > 0.7f => 1.1f,

        _ => 1.0f  // 默认不修改
    };
}

4.1 Memory 类新增字段（MemoryContracts.cs）

字段类型说明谁写入
simDayint 记忆创建时的仿真天数（第1天=1）。用于按天检索，区分"今天发生"vs"昨天发生"。 MemoryModule.StoreMemory（自动注入）

4.2 MemoryModule.StoreMemory 自动注入逻辑

// MemoryModule.cs — StoreMemory 方法末尾增加
private void InjectTimeAndPersonality(Memory m)
{
    // ① 注入时间上下文
    if (_timeModule != null)
    {
        m.simDay     = _timeModule.CurrentDay;
        m.simHour    = _timeModule.CurrentHour;
        m.timePeriod = _timeModule.CurrentPeriod.ToString();

        // 自动追加时段 tag，让 Recall 时能按时段过滤
        if (!m.tags.Contains(m.timePeriod))
            m.tags.Add(m.timePeriod);
    }

    // ② 注入人格标注并修正重要度
    if (_personalitySystem != null)
    {
        // 根据人格调整 importance 权重
        float modifier = _personalitySystem.GetImportanceModifier(m.kind);
        m.importance = Mathf.Clamp01(m.importance * modifier);

        // 记录人格标注到 detail，供反思模块分析
        string pCtx = _personalitySystem.GetPersonalityTag();
        if (!string.IsNullOrWhiteSpace(pCtx))
            m.personalityContext = pCtx;
    }
}
